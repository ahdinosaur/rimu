# Expressions

## Atoms

### Null

```rimu
null
```

### Boolean

```rimu
true
```

```rimu
false
```

### Number

> All numbers in Rimu are ["Big" Decimals](https://crates.io/crates/rust_decimal), not floats or integers.

```rimu
10
```

```rimu
10.99
```

### String

```rimu
"hello world"
```

> Escape quotes within strings to represent them literally.

```rimu
"they said \"hello world\""
```

### Function

```rimu
(a, b) => a + b
```

### Identifier

> Identifiers must conform to regex: `^[a-zA-Z_][a-zA-Z0-9_]*$`

```rimu
hello
```


## Collections

### List

```rimu
["hello", null, "world"]
```

### Object

> Unquoted keys must conform to [Identifier](#Identifier) regex: `^[a-zA-Z_][a-zA-Z0-9_]*$`
>
> Quoted keys can be any valid Unicode.


```rimu
{ a: "apple", b: "bear" }
```

```rimu
{ "a": "apple", "b": "bear" }
```

## Operators

### Unary

> Operand must be a number.

#### `-` (negate)

```rimu
-10
```

#### `!` (not)

> Operand must be a boolean.

```rimu
!is_done
```

### Binary

#### `+` (add)

> Operands must be numbers, strings, or lists.

```rimu
10 + 2
```

```rimu
"hello" + "world"
```

```rimu
[1, 2] + [3, 4]
```

#### `-` (subtract)

> Operands must be numbers.

```rimu
10 - 2
```

#### `*` (multiply)

> Operands must be numbers.

```rimu
10 * 2
```

#### `/` (divide)

> Operands must be numbers.

```rimu
10 / 2
```

#### `>` (greater than)

> Operands must be numbers.

```rimu
10 > 2
```

#### `>=` (greater than or equal)

> Operands must be numbers.

```rimu
10 >= 2
```

#### `<` (less than)

> Operands must be numbers.

```rimu
10 < 2
```

#### `<=` (less than or equal)

> Operands must be numbers.

```rimu
10 <= 2
```

#### `==` (equal)

```rimu
true == false
```

#### `!=` (not equal)

```rimu
true != false
```

#### `&&` (and)

```rimu
true && false
```

#### `||` (or)

```rimu
true || false
```

#### `^` (xor)

> Operands must be booleans.

```rimu
true ^ false
```

#### `%` (remainder)

> Operands must be numbers.

```rimu
10 % 2
```

### Call

> Value being called must be a function.

```rimu
add(1, 2)
```

### Get Index

> Container must be list or string or object.

```rimu
list[2]
```

```rimu
list[-2]
```

```rimu
string[2]
```

```rimu
object["a"]
```

### Get Key

> Container must be object

```rimu
object.a
```

### Get Slice

> Container must be list or string.

```rimu
list[2:5]
```

```rimu
string[2:5]
```

## Notes

### Truthiness

Everything is truthy except `false` and `null`.

### Orderedness

Only numbers are ordered.

### No statements

Every expression is evaluated to a [value](./values).

### No type coercion

No implicit type coercions.

For example, the following is an error:

```rimu
2 + "pies"
```
